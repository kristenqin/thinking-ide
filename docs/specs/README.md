## Maintenance Rule

These files are the repo's spec layer, not the operational status layer.

1. Update `PROJECT_STATUS.md`, `risk-register`, and `traceability-matrix` first for ordinary implementation progress.
2. Update a spec in this folder when a slice changes the contract that spec claims to define, or when leaving it stale would mislead future work.
3. Use [document-system-map.md](/Users/qyx/Desktop/project/thinking-ide/docs/document-system-map.md) and [document-sync-policy.md](/Users/qyx/Desktop/project/thinking-ide/docs/document-sync-policy.md) to decide when a spec change is required.
